**1. Entidades**

Son cosas o personas de las que guardamos datos.

Ejemplo: Alumno, Curso.

Cada entidad tiene atributos (nombre, edad...).

**2. Relaciones**

Son los vínculos entre entidades.

Ejemplo: Un alumno se matricula en un curso.

**3. Cardinalidad**

Indica cuántos elementos de una entidad se relacionan con cuántos de otra:

1 a 1

1 a muchos

Muchos a muchos

✔ El modelo ER sirve para planificar una base de datos, mostrando qué datos habrá y cómo se conectan.
