Los **lenguajes de muy bajo nivel** son aquellos que se acercan más directamente al hardware de una computadora, esto implica que son específicos para una arquitectura de procesador.

**Ejemplo:** El Código de Máquina (o Código Binario) es el lenguaje más fundamental, compuesto únicamente por secuencias de '0' y '1'.

El **Lenguaje Ensamblador** es el siguiente nivel por encima del código de máquina.
Aunque es complejo, es útil porque permite el control más granular y preciso del hardware

### **Paradigma Estructurado**
El paradigma estructurado es un enfoque de programación que busca mejorar la claridad, calidad y tiempo de desarrollo del software.
Se utiliza para organizar el flujo del programa de manera lógica y secuencial, facilitando la comprensión y el mantenimiento del código.

El paradigma Orientado a Objetos organiza el diseño del software alrededor de los objetos en lugar de las funciones y la lógica.
Una de sus ventajas es que trae grandes beneficios al desarrollo de proyectos grandes.

### **Python como Lenguaje Multiparadigma**
Elegimos *Python* como ejemplo de lenguaje multiparadigma. Un **lenguaje multiparadigma** es aquel que permite al programador usar más de un paradigma de programación dentro del mismo proyecto.

El programador puede elegir entre diferentes paradigmas dependiendo del tipo de problema que necesita resolver y de la organización que prefiera darle a su código.
En la práctica, muchos proyectos de Python combinan estos estilos, aprovechando lo mejor de cada paradigma para diferentes partes del sistema.
