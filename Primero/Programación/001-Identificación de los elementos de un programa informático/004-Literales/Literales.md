Cuando identificamos un literal numérico en un archivo .py, observamos un valor que se escribe directamente, tal como es, en el código fuente. En este caso, al ser numérico, solo representará un número, no un dato.

Un literal  numérico es una notación para representar un valor fijo en el código. A diferencia de una variable, un literal no cambia; siempre representa ese valor exacto.

Cuando indentificamos un literal de cadena, como es el archivo 001-literales de cadena.py estamos viendo un valor fijo que se escribe directamente en el código.

Lo que lo identifica como un literal de cadena (a diferencia de un número o una variable) son las comillas dobles que encierran el texto.

Para hacer una operacion de variables, hay nombrarlas con números:

Variable para la edad (Literal Entero)
edad_natacionista = 25

Variable para el precio (Literal Decimal/Flotante)
precio_entrada = 10.99

Una vez son nombradas con sus respectivos datos numéricos, procedemos a la operación:

Suma de la edad y el precio. 
El resultado será un valor decimal
total = edad_natacionista + precio_entrada

print("El valor total (edad + precio) es: (total)")

Una vez tenemos la operacion de variables, procedemos a la **concatenación de variables**:

edad_competidor = 28 
nombre = "Javier"

mensaje = "Hola, soy un competidor llamado " + nombre + " y mi edad es " + (edad_competidor)

print(mensaje)

Con todas estas operaciones, tanto literales como numéricas, hemos tecleado datos fijos, saber cómo definir y nombrar espacios de memoria para esos datos, y cómo combinar texto y números.
